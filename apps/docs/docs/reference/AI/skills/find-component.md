# find-component

Before building a new page-builder section, check whether one already exists. The skill searches the component library two ways — by a **text description** of what you need, or by a **screenshot** of the UI — and tells you what to reuse or whether to build fresh. It's read-only: it inspects and recommends, never edits.

## Use it when

- You're about to add a section and want to avoid duplicating an existing one.
- You have a design or screenshot and need to know which component renders that pattern.
- You want to reuse a shared row/child component instead of rebuilding it.

## What it helps solve

- Answers "do we already have this section?"
- Helps choose an existing section instead of building a duplicate.
- Identifies the closest match for a screenshot or design request.
- Points out when an existing section is close but not quite enough.

If nothing fits, the next step is [create-content-component](./create-content-component.md).
